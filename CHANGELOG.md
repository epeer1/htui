# Changelog

## [0.4.0] - 2026-04-17

### Added
- `htui mcp` — MCP (Model Context Protocol) stdio server exposing 8 tools: `htui_exec`, `htui_run`, `htui_tail`, `htui_get`, `htui_search`, `htui_list`, `htui_kill`, `htui_summary`.
- `htui watch` — live TUI that streams card activity from the local `htui mcp` server over a workspace-scoped socket.
- `htui init` now writes `.vscode/mcp.json` (merges with existing file, preserves indent and other servers), adds an `htui:watch` npm script, and renders agent instructions that lead with the MCP section.
- `--yes` / `-y` / `--no-prompt` flag on `htui init` for non-interactive installs.
- `HTUI_NO_ANIM=1` disables watch-mode animations.

### Fixed
- Windows: `htui_exec` with a timeout now kills the full child-process tree via `taskkill /T /F` (no orphaned grandchildren).
- `htui_get` / `htui_tail` results now report `truncated: true` when the ring buffer has already evicted lines, not only when the caller's `maxLines` is exceeded.
- `htui_exec` timeout results now also include `timedOut: true` and `killedBy: 'timeout'` alongside `status: 'timeout'`.

### Changed
- Agent instruction files generated by `htui init` now lead with MCP tools; `htui exec` is documented as a fallback and `htui --api` as legacy.
